## Decisions Made This Session

1. **Flask POST + fw CLI subprocess for write actions**
   - Why: Single source of truth — CLI handles all validation, ID generation, file manipulation. Flask routes are thin wrappers. No code duplication.
   - Alternatives rejected: Direct Python YAML manipulation, JavaScript fetch + REST API

2. **Standard form POST + redirect over HTMX partial swaps**
   - Why: Simpler implementation, full page refresh ensures consistent state, HTMX partial would need fragment endpoints
   - Alternatives rejected: HTMX hx-post with fragment swap

3. **markdown2 with list normalization regex**
   - Why: markdown2 already installed, Markup() avoids XSS. Regex to insert blank lines before list markers fixes rendering without modifying source data.
   - Alternatives rejected: markdown library, manual regex replacement

## Things Tried That Failed

1. **POST without CSRF token** — Flask CSRF middleware returns 403. Fixed by adding hidden `_csrf_token` field with `{{ csrf_token() }}` to all forms.
2. **markdown2 without list normalization** — Task markdown often lacks blank lines before bullet lists, causing markdown2 to treat them as paragraph text. Fixed with regex: `re.sub(r"([^\n])\n(- )", r"\1\n\n\2", text)`.
