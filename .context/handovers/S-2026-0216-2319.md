## Decisions Made This Session

1. **D-019: Tags over hierarchy for task grouping**
   - Why: Flat tags are simpler, more flexible, avoid taxonomy design problem. Epics/features add brittle coupling for single-agent work.
   - Alternatives rejected: Formal backlog register, epic/feature/task hierarchy, sprint planning

2. **D-020: Passive git metrics over manual estimation**
   - Why: Manual story points meaningless for AI agents. Git data is objective and already available. Learning loop via `fw metrics predict`.
   - Alternatives rejected: Story points, token-based estimation, time-based estimation

3. **Deliberate application counting for graduation pipeline**

   - Alternatives rejected: Keyword similarity scoring, fuzzy text matching
