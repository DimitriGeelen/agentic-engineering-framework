## Decisions Made This Session

1. **Shared tooling model (Option B)**
   - Why: Framework is tooling, not app code — shouldn't pollute project repos
   - Alternatives rejected: In-repo (pollutes), companion repo (split context), gitignored (loses traceability)

2. **`fw` CLI wrapper + `.framework.yaml` config over symlinks/env vars**
   - Why: Single resolution strategy, portable, debuggable, solves PROJECT_ROOT
   - Alternatives rejected: Symlinks (D4 violation, silent breakage), env vars (hidden config), absolute paths (machine-specific)

3. **Pre-push hook blocks on missing audit (exit 1)**
   - Why: D2 "no silent failures" — old behavior silently allowed push
   - Alternatives rejected: Warning only (too weak), silent allow (D2 violation)

4. **`${PROJECT_ROOT:-$FRAMEWORK_ROOT}` pattern for backward compatibility**
   - Why: Agents work both as shared tooling (PROJECT_ROOT set externally) and standalone (fallback to ../../)
   - Alternatives rejected: Hard requirement for external PROJECT_ROOT (breaks current usage)

5. **harvest.sh for cross-project learning feedback**
   - Why: D1 antifragility requires learning from all projects, not just self-referential
   - Alternatives rejected: Manual promotion (won't happen), shared patterns.yaml (merge conflicts)

6. **Auto-trigger healing on `issues` status**
   - Why: P-002 structural enforcement over agent discipline — healing shouldn't depend on someone remembering
   - Alternatives rejected: Manual invocation only (current, unreliable under stress)

7. **Version pinning in .framework.yaml**
   - Why: D2 reliability — project must know which framework version it expects
   - Alternatives rejected: No versioning (breaks silently when framework evolves)

8. **Add practices.yaml as structured data**
   - Why: Knowledge pyramid gap — healing loop can't query practices, harvest can't compare them
   - Alternatives rejected: Keep markdown only (not machine-queryable)

9. **Framework evaluation spawned as subagent**
   - Why: Independent critical analysis against all 4 directives, found PROJECT_ROOT showstopper
   - Result: Symlinks rejected, wrapper CLI recommended, silent failures flagged
