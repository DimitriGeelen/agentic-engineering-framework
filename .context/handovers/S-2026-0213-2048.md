## Decisions Made This Session

1. **Git agent absorbs T-003 and T-004**
   - Why: Coherent tool for all git enforcement rather than scattered scripts
   - Alternatives rejected: Separate bypass-log script, standalone pre-commit hook

2. **Use commit-msg hook (not pre-commit)**
   - Why: Validates message format, not file content; more appropriate for task ref check
   - Alternatives rejected: pre-commit (runs before message exists)

3. **Audit saves history to YAML files**
   - Why: Enables trend detection without database, human-readable
   - Alternatives rejected: SQLite (overkill), JSON (less readable)

4. **3+ occurrences triggers practice candidate**
   - Why: Balance between noise (too few) and missing patterns (too many)
   - Alternatives rejected: 2 (too noisy), 5 (too slow to detect)

5. **Tier 0 violations are FAIL not WARN**
   - Why: Consequential actions must never bypass task requirement
   - Alternatives rejected: WARN (too weak for Tier 0)
