## Decisions Made This Session

1. **Scan engine lives in `web/watchtower/`, not `lib/` or `agents/`**
   - Why: `lib/` is shell-only (D4); agents have INITIATIVE but scan has AUTHORITY; `web/` already has Python + YAML deps
   - Alternatives rejected: `lib/watchtower/` (language boundary), `agents/scan/` (wrong authority level)

2. **Scan is stateless — no dismiss/undo in scan YAML**
   - Why: Prevents parallel tracking system competing with task system as gravitational center
   - Alternatives rejected: Storing dismiss state in scan YAML (creates scan-specific state outside task system)

3. **`framework_acted` renamed to `framework_recommends`**
   - Why: Scan never auto-mutates. All mutations go through existing Tier 1 commands. Preserves "nothing without a task."
   - Alternatives rejected: Auto-acting framework (violates tier separation, creates accountability gap)

4. **Scan supplements handover, never replaces**
   - Why: Handover has tacit knowledge (failed approaches, gotchas) that scan can't compute
   - Alternatives rejected: Scan replacing LATEST.md (loses strategic context)

5. **Feedback loop is mandatory (D1)**
   - Why: Scan must be antifragile to its own mistakes. Track approval rate per recommendation type.
   - Alternatives rejected: No feedback (one-directional scan violates D1)

## Things Tried That Failed

1. **Initial design had `framework_acted` with auto-mutations** — Architect reviewer caught that this creates accountability gap and violates "nothing without a task". Changed to `framework_recommends`.

2. **Proposed scan replaces handover reading** — Agent Dev reviewer caught that handover contains tacit knowledge (failed approaches, warnings) that scan can't compute. Changed to "supplements."
