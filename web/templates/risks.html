{{ page_title }}

Three-register assurance model for framework governance. Risks are forward-looking (what could go wrong), issues are backward-looking (what did go wrong), gaps are present-tense (spec vs reality).

About this page

Scoring model: Likelihood (1-5: Rare → Almost certain) × Impact (1-5: Negligible → Severe) = Score (1-25).
Ranking thresholds: LOW 1-4, MEDIUM 5-9, HIGH 10-16, URGENT 17-25.

Control status: CLOSED = mitigated, BUILD = control designed but not yet built, ACK = acknowledged / accepted risk, WATCH = monitoring for change.

Constitutional Directives: Risk management is driven by D1 (Antifragility — failures are learning events) and D2 (Reliability — auditable, no silent failures).

Artifacts:

  • Genesis discussion — how the assurance model was designed (T-194)
  • Risk landscape analysis — methodology, evidence mining, cross-cutting patterns
  • Control register design — schema decisions, control inventory, design adequacy

Data sources: .context/project/risks.yaml, .context/project/issues.yaml, .context/project/gaps.yaml
