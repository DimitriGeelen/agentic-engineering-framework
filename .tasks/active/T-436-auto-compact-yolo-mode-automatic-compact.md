# T-436: Auto-compact YOLO mode: automatic compact → resume → autonomous proceed

## Problem Statement

Currently, context compaction and resume are manual operations. The human must:
1. Notice context is getting high (or wait for budget-gate to block)
2. Run `/compact` manually
3. Run `/resume` manually after compaction
4. Decide what to work on next

This breaks flow and requires human presence at context transitions. The framework's token
management infrastructure is now mature:
- **PreCompact hook** auto-generates handover before compaction (T-111/T-188)
- **SessionStart:compact hook** re-injects context after compaction (T-188)
- **Budget-gate** blocks source edits at critical context (T-139)
- **Auto-restart wrapper** (`claude-fw`) detects handover signal and restarts (T-179)
- **checkpoint.sh** monitors token usage continuously (T-136)

After 150+ sessions of reliable operation, the question is: can we close the loop entirely?
"YOLO mode" = enable `autoCompactEnabled: true`, let the existing hooks handle the
transition, and have the agent autonomously continue work after resume — executing
everything that doesn't require human validation.

**Why now:** The framework is approaching external adoption. Manual compact/resume is a
usability friction point. If the safety net (hooks, handovers, budget-gate) is solid,
removing the manual step improves D3 (Usability) without sacrificing D2 (Reliability).

**Why "YOLO":** It's the human's term for "I trust the system enough to let it run
unsupervised through context transitions." Not reckless — earned confidence.

## Assumptions

- **A1:** `autoCompactEnabled: true` in `~/.claude.json` triggers Claude Code's built-in
  auto-compaction at the same threshold as manual `/compact` (~190K tokens)
- **A2:** PreCompact hook still fires when auto-compact triggers (not just manual `/compact`)
- **A3:** SessionStart:compact hook fires after auto-compaction (same as manual)
- **A4:** The handover generated by PreCompact contains enough context for autonomous
  continuation (no human needed to fill in [TODO] sections)
- **A5:** The agent can reliably determine what work requires human validation vs. what
  can proceed autonomously (Autonomous Mode Boundaries in CLAUDE.md already define this)
- **A6:** Auto-restart wrapper (`claude-fw`) is compatible with auto-compact
  (or auto-compact makes it unnecessary)
- **A7:** Context budget tracking resets correctly after auto-compaction
  (L-049 documented a budget-gate deadlock after compaction — verify this is resolved)

## Technical Constraints

- `autoCompactEnabled` is a Claude Code internal setting in `~/.claude.json` — behavior
  may differ from manual `/compact` (investigate in Spike 1)
- D-027 decision explicitly disabled auto-compact for good reasons — this inception must
  address every concern from D-027 or explain why they no longer apply
- PreCompact hook timeout is 600s — must complete handover within that window
- SessionStart hooks inject context but cannot trigger `/resume` skill directly — the
  resume must be structural (hook output), not skill-based
- The `claude-fw` auto-restart wrapper (T-179) may interact/conflict with auto-compact
  (both handle context exhaustion, different mechanisms)

## Scope Fence

**IN scope:**
- Verifying hook behavior with auto-compact enabled
- Designing autonomous continuation logic after resume
- Human-gate deferral mechanism (queue, not skip)
- Risk analysis and mitigation design
- Toggle mechanism for YOLO mode (opt-in, not default)
- Addressing D-027 concerns

**OUT of scope:**
- Full implementation (that's build tasks from this inception)
- Changing the Authority Model (human still sovereign)
- Removing any existing safety mechanisms (budget-gate, tier-0, task-gate)
- Multi-agent YOLO (one agent at a time)

## Go/No-Go Criteria

**GO if:**
- PreCompact and SessionStart hooks fire reliably on auto-compact (A2, A3 confirmed)
- Budget reset works after auto-compact (no L-049 deadlock)
- D-027 concerns are addressed or no longer apply
- Autonomous continuation can respect all existing gates (task-gate, tier-0, sovereignty)
- Risk analysis shows no catastrophic failure modes without mitigation
- Implementation can be phased: YOLO-lite (auto-compact + auto-resume, human chooses work)
  before YOLO-full (auto-compact + auto-resume + autonomous continuation)

**NO-GO if:**
- Hooks don't fire on auto-compact (A2/A3 fail — core assumption broken)
- Budget reset deadlocks persist (L-049 unresolved)
- D-027 concerns remain valid and unmitigable
- Autonomous continuation cannot respect sovereignty without new enforcement mechanisms
- Risk of context loss or work duplication is high and unmitigable
